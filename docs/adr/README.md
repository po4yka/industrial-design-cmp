# Architecture Decision Records

Durable records of non-obvious decisions. One file per decision. Once accepted, edits are appended as new ADRs that supersede, rather than rewriting history in place.

| # | Title | Status |
|---|-------|--------|
| [0001](0001-component-maturity-labels.md) | Component maturity labels | Accepted |
| [0002](0002-no-animations-by-default.md) | No spring easing; 200–300ms ease-out only | Accepted |

## Template

```markdown
# ADR NNNN: Title

## Status
Accepted | Proposed | Superseded by ADR-NNNN

## Context
What forces are at play?

## Decision
What we decided.

## Consequences
What becomes easier or harder as a result.
```
